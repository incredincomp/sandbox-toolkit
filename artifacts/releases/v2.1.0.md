# sandbox-toolkit v2.1.0

Release date: 2026-03-15

## Highlights

- Added read-only maintenance and inspection command surfaces for repeatable operations:
  - `-ListTools`
  - `-ListProfiles`
  - `-DryRun`
  - `-Validate`
  - `-CheckForUpdates`
  - `-CleanDownloads`
- Added custom profile support (`custom-profiles.local.json`) plus runtime tool overrides:
  - `-AddTools`
  - `-RemoveTools`
- Added saved sessions/templates:
  - `-SaveTemplate`
  - `-ListTemplates`
  - `-ShowTemplate`
  - `-Template`
- Added curated first-wave tool/profile expansion:
  - tools: `dependencies`, `api-monitor`, `procdot`, `visual-studio-community`, `windows-sdk`
  - built-in profiles: `triage-plus`, `reverse-windows`, `behavior-net`, `dev-windows`

## Precedence Rules

Profile/tool selection precedence remains explicit and deterministic:

1. base profile selection
2. custom profile resolution (`add_tools`, then `remove_tools`) when selected
3. runtime overrides (`-AddTools`, then `-RemoveTools`)

Template-driven runs additionally apply template defaults before explicit runtime overrides.

## Hardening and Contracts

- Integration characterization coverage was expanded for:
  - dry-run/validate combinations (built-in/custom/add/remove paths)
  - invalid profile/tool inputs
  - list-mode state reflection
  - cleanup-scope confinement
- Error handling and exit code behavior are standardized around deterministic non-zero failures for fatal input/config issues.
- `-CheckForUpdates` is read-only and does not mutate `tools.json`.

## Safety and Scope Notes

- `api-monitor`, `procdot`, `visual-studio-community`, and `windows-sdk` are intentionally represented as manual/advanced where unattended automation is not reliable in this pass.
- REMnux and VirusTotal uploader bundling are intentionally deferred.
